# 굿애프터눈 (Good Afternoon.) — 기획 컨텍스트 프롬프트

## 이 프롬프트의 목적
이 문서는 "굿애프터눈" 서비스 기획의 현재 상태를 다른 에이전트가 이어받기 위한 맥락 문서다.
기획자와의 대화를 통해 확정된 내용, 현재 미결 사항, 그리고 다음 작업 방향을 담고 있다.

---

## 1. 서비스 정의

**이름:** 굿애프터눈 (Good Afternoon.)

**정체:** 경제 개념 잠금해제 서비스. 강의도, 학습 앱도 아님.
가장 가까운 레퍼런스는 **Duolingo + 마인크래프트 업적 시스템의 교차점**.

**네이밍 의도:**
- 제롬 파월(Jerome Powell)이 FOMC 기자회견을 시작할 때 하는 말 "Good afternoon."에서 차용
- 뉴비: 그냥 인사처럼 읽힘 → 자연스럽게 진입
- 아는 사람: FOMC 바이브 즉각 픽업 → 씩 웃고 시작
- 이름의 의미는 온보딩에서 절대 설명하지 않음
- 회수 지점: 철 티어 이스터에그 문항에서 처음으로 정면 해설됨

**핵심 문제의식:**
한국 교육과정에 실생활 경제 내용이 전무하다.
화폐·이자·신용 같은 기초 개념을 학교에서 배우지 못한 채 성인이 된다.

**타깃:**
- 1차: 중학생·고등학생
- 2차: 성인 (늦게 시작하는 것도 환영하는 톤)
- 숨겨진 타깃: 파월 밈을 아는 경제/투자 관심층

## 2. 게임 철학 (최상위 방향성)

**Easy to learn, Hard to master**

기획자가 명시한 핵심 철학이며, 현재 구조에 대한 문제의식이 있는 상태다.

**현재 진단 (기획자 본인의 의문):**

```
Easy to learn  ████░░░░░░  아직 마찰이 있음
Hard to master ░░░░░░░░░░  레이어 자체가 없음
```

- Easy to learn 문제: 첫 문제까지 가는 단계가 너무 많음. 언어 톤이 여전히 교과서 느낌.
- Hard to master 문제: 107문제 다 풀면 끝. 마스터리의 정의 자체가 없음.

**이 철학을 실현하기 위한 미결 과제가 현재 가장 중요한 작업이다.**

## 3. 확정된 구조

### 3.1 전체 구조: 트렁크 + 브랜치 (2축)

```
수직축 — 트렁크 (메인 퀘스트, 순서 있음)
  🪨 돌 티어: 화폐와 교환        12문제
  🪵 나무 티어: 수입과 저축       10문제
  ⚙️ 철 티어: 신용과 이자        8문제 + 🥚 이스터에그 1문제
         ↓
  [철 티어 클리어 시 브랜치 전체 해금]
         ↓
수평축 — 브랜치 (사이드 퀘스트, 자유 탐험)
  💰 투자       주식·채권·ETF·복리·리스크·리밸런싱     6개념 16Q
  🏠 자산       자산/부채·동산/부동산·감가상각·레버리지   5개념 12Q
  🛡 절세       세금기초·ISA·IRP·연금·연말정산          5개념 12Q
  🧠 습관       행동경제학·FIRE·소비습관·현재편향·자동화   5개념 11Q
  📈 경제현상   인플레·금리·환율·경기사이클·실업·GDP     6개념 15Q
  📚 역사       대공황·IMF·닷컴버블·2008·반도체사이클   5개념 11Q
```

**총 문제 수:** 트렁크 30Q + 브랜치 77Q = **107Q** (이스터에그 1Q 별도)

**브랜치 간 느슨한 추천 경로 (강제 아님):**
- 투자 → 절세
- 경제현상 → 역사
- 자산 → 투자

### 3.2 개념 카드 구조

카드 한 장의 구성:
1. 핵심 한 줄
2. 왜 나한테 중요한가 (추상적 정의 없이 내 삶과 연결)
3. 실생활 예시 1개
4. 열쇠 찾기 (퀴즈 1문제)

### 3.3 퀴즈 설계 원칙

- 문제 수: 1개 (복수 문제 = 공부 느낌)
- 형식: 객관식 2~3지선다
- 오답 처리: 벌칙 없음, 1.8초 후 재시도
- 정답 피드백: "열렸다!" + 다음 해금 예고
- 버튼 이름: "퀴즈" 아님, "열쇠 찾기"

### 3.4 트리거 방식

위젯으로 유도만 하고, 실제 해제는 "배우고 싶을 때 열어서 퀴즈로 해제"하는 방식으로 단순화. 행동 체크인 검증 없음.

## 8. 현재 미결 사항 및 다음 과제

### 8.1 최우선 과제 (Easy to learn, Hard to master 실현)

**Easy to learn 측 개선 필요:**

현재 문제점:
- 첫 문제까지 가는 단계: 앱 설치 → 온보딩 → 타깃 선택 → 허브 → 카드 → 읽기 → 퀴즈 (너무 많음)
- 카드 언어 톤이 여전히 교과서 냄새 ("교환 매개체의 탄생" 같은 표현)

개선 방향 후보:
- 첫 문제를 온보딩 안으로 당기기 (문제 먼저 → 앱 정체 자연스럽게 드러남)
- 카드 제목을 질문형으로 전환 ("화폐란 무엇인가" → "왜 조개껍데기로 물건을 샀을까?")

**Hard to master 레이어 설계 필요:**

현재 없는 것:
- 107문제 다 풀면 끝. 마스터리의 정의 자체 없음.

설계 방향 후보 (결정 안 됨):
- A. 깊이 레이어: 각 개념에 L1/L2/L3 존재. L3는 실제 계산·사례 분석 필요
- B. 연결 마스터리: 브랜치를 넘나드는 크로스 문항 ("금리가 오르면 왜 주가가 내리는가")
- C. 시사 연결: 실제 뉴스 헤드라인을 보고 어떤 개념과 연결되는지 맞추기

### 8.2 기타 미결

- 진행 저장 방식: 로컬스토리지(비계정) vs 계정 기반
- 첫 배포 플랫폼: 모바일 앱 vs 웹 우선
- 브랜치 퀴즈 77개 문항 실제 작성 (초안 없음)
- 철 티어 클리어 → 브랜치 해금 연출 미설계
- 메인 허브 화면 완성본 미설계
- 이스터에그 뱃지 2·3번 조건 미정
- 수익 모델 미검토

## 10. 에이전트에게 전달하는 작업 지시

이 프롬프트를 받은 에이전트는 다음을 숙지한다:

1. **최상위 철학은 "Easy to learn, Hard to master"** — 모든 설계 결정은 이 기준으로 판단한다.

2. **현재 구조는 뼈대만 있는 상태** — 진입 마찰과 마스터리 레이어 모두 미완성임을 인지한다.

3. **기획자의 톤을 유지한다** — 공부 앱이 아님. 경제 용어를 무심하게 툭 던지는 느낌. 설명 과잉 금지.

4. **이스터에그 구조를 훼손하지 않는다** — "Good Afternoon."의 의미는 철 티어 이전에 절대 정면 설명하지 않는다.

5. **확정된 것과 미결인 것을 구분한다** — 섹션 3~6은 확정, 섹션 8은 미결. 확정된 내용을 임의로 바꾸지 않는다.

다음 작업 중 하나를 지시받을 수 있다:
- Easy to learn 개선: 첫 진입 플로우 재설계
- Hard to master 설계: 마스터리 레이어 구조 결정 및 목업
- 브랜치 퀴즈 문항 77개 초안 작성
- 메인 허브 화면 목업
- 철 티어 해금 연출 목업
