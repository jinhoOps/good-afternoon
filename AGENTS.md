# AGENTS.md

이 문서는 굿애프터눈 프로젝트에서 작업하는 에이전트와 개발자를 위한 공통 지침입니다. `GEMINI.md`는 Gemini용 harness 인덱스로 유지하고, 이 파일은 도구와 모델에 상관없이 적용되는 작업 기준으로 사용합니다.

## 작업 언어

- 모든 응답, 문서, 주석은 한국어 존댓말을 기본으로 합니다.
- 파일 인코딩은 UTF-8을 유지합니다.
- 사용자를 향한 설명은 짧고 구체적으로 씁니다.

## 프로젝트 미션

굿애프터눈은 공부처럼 느껴지지 않는 경제 개념 잠금해제 서비스입니다. 핵심 철학은 `Easy to learn, Hard to master`입니다.

- Easy to learn: 첫 진입은 장소, 행동, 짧은 로그로 구성합니다.
- Hard to master: 후반부는 복합 판단, 지연 피드백, 실제 경제 시나리오 분석으로 깊이를 만듭니다.
- 철 티어 이전에는 `Good Afternoon.` 이름의 의미를 직접 설명하지 않습니다.

## 문서 우선순위

1. `PROJECT_CONTEXT.md`: 서비스 정의와 확정된 기획 불변량
2. `docs/ROADMAP.md`: 개발 순서와 다음 구현 후보
3. `DESIGN.md`: Good Afternoon 디자인 기준
4. `harness/30_mastery_spec.md`: 마스터리 레이어 상세 설계
5. `harness/`: 검증, 무결성, SOP 세부 규칙
6. `README.md`: 새 개발자가 보는 실행 안내

작업 전에는 관련 문서를 먼저 읽고, 문서 간 충돌이 있으면 `PROJECT_CONTEXT.md`의 확정 구조를 우선합니다.

## 개발 원칙

- 현재 구현의 소스 오브 트루스는 `src/`입니다.
- Pre-Cyan 기준 구현은 `src/pre-cyan-village/`에서 관리합니다.
- Vite + TypeScript를 기준으로 개발하고, `npm run dev`, `npm test`, `npm run build` 흐름을 사용합니다.
- `dist/`는 빌드 출력물이며 커밋하지 않습니다.
- 이전 `app/` 경로는 새 구현의 기준선이 아닙니다. 필요한 경우 과거 구조 확인 용도로만 봅니다.
- 파일은 책임이 보이도록 작게 나눕니다.
- 기존 루트 HTML 데모는 참고 자료입니다. 새 구현의 소스로 복사하거나 덮어쓰지 않습니다.
- 새 기능은 `src/` 하위에 독립적인 TypeScript 모듈 또는 앱 슬라이스로 둡니다.

## 디자인 원칙

- 따뜻한 연크림 캔버스, 차콜 텍스트, 브론즈 골드 포인트를 기본으로 합니다.
- Pre-Cyan 첫 경험은 카드 체크리스트가 아니라 노드와 길이 있는 마을 지도입니다.
- 긴 설명, 점수, 학습 완료 문구는 첫 경험 전면에 두지 않습니다.
- 보상은 숫자보다 “길이 열렸다”는 감각으로 표현합니다.
- 복권방 뒤 어두운 골목은 투자 브랜치의 숨은 씨앗으로 유지합니다.

## 작업 루프

모든 변경은 다음 순서로 진행합니다.

1. Research: 현재 문서와 코드를 읽습니다.
2. Strategy: 바꿀 파일과 검증 기준을 정합니다.
3. Execution: 작고 명확한 변경을 적용합니다.
4. Validation: 문자열 스캔, 브라우저 스모크 테스트, 모바일 뷰포트 점검 중 해당하는 검증을 수행합니다.

## 검증 기준

문서 변경:
- 오래된 미래형 설명이 남지 않았는지 확인합니다.
- 새 문서가 README 또는 관련 인덱스에서 발견 가능한지 확인합니다.
- `PROJECT_CONTEXT.md`, `docs/ROADMAP.md`, `DESIGN.md`가 서로 충돌하지 않는지 확인합니다.

프론트엔드 변경:
- 브라우저 콘솔 오류가 없어야 합니다.
- 모바일 390x844 기준으로 가로 스크롤과 텍스트 넘침이 없어야 합니다.
- 저장 상태가 새로고침 후 유지되고, 깨진 저장 데이터는 안전하게 초기화되어야 합니다.
- Pre-Cyan 첫 경험에 `필수 기초 단어 놀이터`, `이해함`, `점수`, `학습 완료` 같은 표현을 넣지 않습니다.

## 금지 사항

- 사용자가 요청하지 않은 대규모 프레임워크 전환
- 루트 HTML 데모를 새 구현 소스로 취급하는 것
- 철 티어 이전에 앱 이름의 의미를 설명하는 것
- Pre-Cyan에 마스터리 레이어의 복잡도를 끌어오는 것
- 사용자 변경사항을 임의로 되돌리는 것
