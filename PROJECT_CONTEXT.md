# 굿애프터눈 (Good Afternoon.) 서비스 기획 문서

## 1. 서비스 정의
- **이름**: 굿애프터눈 (Good Afternoon.)
- **정체**: 경제 개념 잠금해제 서비스 (강의도, 학습 앱도 아님. Duolingo + 마인크래프트 업적 시스템 교차점)
- **네이밍 의도**:
  - 제롬 파월(Jerome Powell)이 FOMC 기자회견을 시작할 때 하는 말 "Good afternoon."에서 차용
  - 뉴비: 그냥 인사처럼 읽힘 → 자연스럽게 진입
  - 아는 사람: FOMC 바이브 즉각 픽업 → 씩 웃고 시작
  - 이름의 의미는 온보딩에서 절대 설명하지 않음. 철 티어 이스터에그 문항에서 정면 해설됨
- **핵심 문제의식**: 한국 교육과정에 실생활 경제 내용이 전무.
- **타깃**:
  - 1차: 중학생·고등학생
  - 2차: 성인 (늦게 시작하는 것도 환영하는 톤)
  - 숨겨진 타깃: 파월 밈을 아는 경제/투자 관심층

## 2. 게임 철학 (최상위 방향성)
**Easy to learn, Hard to master**

현재 진단:
- **Easy to learn**: 아직 마찰이 있음. (첫 문제까지 가는 단계가 너무 많음. 언어 톤이 교과서 느낌)
- **Hard to master**: 레이어 자체가 없음. (107문제 다 풀면 끝. 마스터리의 정의 자체가 없음)

## 3. 확정된 구조
### 3.1 전체 구조: 트렁크 + 브랜치 (2축)
- **수직축 (트렁크 / 메인 퀘스트)**:
  - 🐣 **Pre-Cyan 티어**: 첫 모험 마을 (0단계)
    - *대상: 경제라는 단어에 부담을 느끼거나, 경제활동 경험이 알바/용돈 수준인 입문자*
    - *성격: 단어 카드나 퀴즈가 아니라, 현실 동네를 닮은 게임 마을을 둘러보고 만져보며 적응하는 첫 지역*
    - *흐름: 내 방에서 시작해 편의점, 버스정류장, 알바처, 은행, 구독함, 복권방을 눌러본다. 사용자가 내 방을 포함해 공개 장소 4개를 방문하면 Cyan 입구가 열린다.*
    - *원칙: 장소 먼저, 개념 나중. 경제 용어와 업적은 첫 경험 전면에 두지 않는다.*
    - *숨김 요소: 복권방 뒤 어두운 골목은 발견한 사람만 나중에 투자 브랜치에서 일부 반응을 먼저 본다.*
  - 🪨 돌 티어 (Cyan 티어): 화폐와 교환 (12문제)
  - 🪵 나무 티어 (Magenta 티어): 수입과 저축 (10문제)
  - ⚙️ 철 티어 (Yellow 티어): 신용과 이자 (8문제 + 이스터에그 1문제)
  - *철 티어 클리어 시 브랜치 전체 해금*

- **수평축 (브랜치 / 사이드 퀘스트)**:
  - 💰 투자 (6개념 16Q)
  - 🏠 자산 (5개념 12Q)
  - 🛡 절세 (5개념 12Q)
  - 🧠 습관 (5개념 11Q)
  - 📈 경제현상 (6개념 15Q)
  - 📚 역사 (5개념 11Q)

- 총 문제 수: 트렁크 30Q + 브랜치 77Q = 107Q (이스터에그 1Q 별도)

### 3.2 개념 카드 구조
- 핵심 한 줄
- 왜 나한테 중요한가 (추상적 정의 없이 내 삶과 연결)
- 실생활 예시 1개
- 열쇠 찾기 (퀴즈 1문제)

### 3.3 퀴즈 설계 원칙
- **문제 수**: 1개
- **형식**: 객관식 2~3지선다
- **오답 처리**: 벌칙 없음, 1.8초 후 재시도
- **정답 피드백**: "열렸다!" + 다음 해금 예고
- **버튼 이름**: "퀴즈" 아님, "열쇠 찾기"

### 3.4 트리거 방식
- 위젯으로 유도만 하고, 실제 해제는 "배우고 싶을 때 열어서 퀴즈로 해제"하는 방식.

## 4. 이스터에그 설계
- **위치**: 철 티어 14번 개념 "기준금리란" 내부
- **문항**: "중앙은행 총재가 기자회견을 시작할 때 항상 하는 말이 있다. 이 인사가 경제에서 갖는 의미에 가장 가까운 것은?"
- **정답**: ③ 전 세계 시장이 귀 기울이는 순간의 시작을 알리는 신호
- **피드백**: "맞아. 'Good afternoon.' — 제롬 파월이 FOMC 기자회견을 시작할 때 하는 그 말. 이 두 단어가 나오는 순간 전 세계 트레이더들이 모니터 앞에 앉아. 이제 이 앱 이름이 왜 그런지 알겠지?"
- **획득 뱃지**: 🎙️ "Good Afternoon."

## 5. 뱃지 시스템 (현재 확정분)
- **티어 클리어**: 🪨 첫 번째 돌, 🪵 나무를 패다, ⚙️ 철의 맛
- **퍼펙트 클리어**: 💯 실수 없이, ⚡ 속전속결, 🗂️ 트렁크 완주
- **이스터에그**: 🎙️ Good Afternoon., 🔮 추후 공개, 🌑 추후 공개

## 6. 온보딩 흐름 (확정)
- 랜딩 → 질문("이 앱 이름이 왜 굿애프터눈인지 알겠어?") → 타깃 선택 → 메인 허브 (개념 지도)

## 7. 톤앤매너

- DM Mono + Playfair Display + Noto Sans KR
- 무심하게 툭 던지는 느낌 ("공부" 단어 배제)
- 코드 주석 스타일 보조 텍스트 (`// 힌트는 저장해뒀어`)

## 8. 현재 미결 사항 및 다음 과제
### 8.1 최우선 과제 (Easy to learn, Hard to master 실현)
- **Easy to learn 개선**: Pre-Cyan을 카드형 대기 공간이 아니라 모바일 세로 한 화면 기준의 첫 모험 마을로 재설계한다. 사용자는 설명을 읽기보다 장소를 만지고, 짧은 로그와 연결 노드 활성화를 통해 조작 방식을 익힌다.
- **Hard to master 설계**: 마스터리 레이어 구조 결정 및 목업 (깊이 레이어 L1/L2/L3, 크로스 문항, 시사 연결 등)

### 8.2 기타 미결
- 진행 저장 방식, 첫 배포 플랫폼, 브랜치 퀴즈 77개 문항 초안, 철 티어 클리어 → 브랜치 해금 연출 목업, 메인 허브 화면 완성본 목업, 이스터에그 뱃지 2·3번 조건 미정, 수익 모델 미검토

## 9. 작업 지시 (Agent Instructions)
- 최상위 철학 "Easy to learn, Hard to master" 기준 판단.
- 공부 앱이 아님. 경제 용어를 무심하게 툭 던지는 느낌.
- 이스터에그 구조 훼손 금지 ("Good Afternoon." 의미 정면 설명 금지).
- 확정된 것(섹션 3~6) 변경 금지.
