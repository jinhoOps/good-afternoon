# Good Afternoon. 개발 로드맵

이 문서는 현재 기획과 데모를 실제 제품 구현 순서로 옮기기 위한 작업 지도입니다. 독자는 이 저장소를 처음 여는 개발자 또는 기획자이며, 읽은 뒤 다음 구현 단위를 선택하고 검증 기준을 잡을 수 있어야 합니다.

## 현재 기준선

- Pre-Cyan 첫 모험 마을의 소스 오브 트루스는 `src/pre-cyan-village/`입니다.
- 현재 기술 기준은 Vite + TypeScript입니다. `dist/`는 빌드로 생성되는 출력물이며 커밋하지 않습니다.
- 기존 `app/` 구조는 점진적으로 폐기합니다. 새 구현의 기준 경로로 사용하지 않고, 필요한 경우 이전 자료 확인 용도로만 봅니다.
- 루트의 HTML 파일들은 기획 검증용 시각 자료입니다. 새 구현의 소스가 아니라, 온보딩 흐름과 기준금리 챌린지의 참고 데모입니다.
- 서비스의 핵심 철학은 `Easy to learn, Hard to master`입니다. Pre-Cyan은 진입 마찰 제거를 맡고, 이후 Cyan 이상 티어와 마스터리 레이어가 깊이를 맡습니다.

## 제품 축

### 1. Entry: Pre-Cyan 첫 모험 마을

역할: 경제 단어를 먼저 설명하지 않고, 현실 동네를 닮은 장소를 눌러보며 조작 방식과 서비스 톤에 적응시킵니다.

현재 상태:
- 내 방에서 시작합니다.
- 공개 장소 4개를 방문하면 Cyan 입구가 열립니다.
- 첫 업적은 Cyan 입구에 들어간 뒤에야 보입니다.
- 복권방 뒤 어두운 골목은 숨은 투자 브랜치 씨앗입니다.

다음 개선 후보:
- 장소별 짧은 사운드 없는 피드백 또는 미세 애니메이션을 추가합니다.
- Cyan 입구 이후 어디로 이어지는지 임시 전환 화면을 만듭니다.
- 접근성 이름, 키보드 조작, 저장 실패 상태를 점검합니다.

### 2. Trunk: Cyan, Magenta, Yellow 메인 티어

역할: 트렁크 개념을 순서대로 잠금해제합니다. 사용자는 강의가 아니라 열쇠를 찾는 느낌으로 화폐, 수입, 저축, 신용, 이자를 통과합니다.

개발 순서:
1. Cyan 티어: 화폐와 교환의 최소 플레이 루프를 만듭니다.
2. Magenta 티어: 수입과 저축의 선택형 상황 문제를 붙입니다.
3. Yellow 티어: 신용과 이자를 다루고, 철 티어 이스터에그 문항으로 앱 이름의 의미를 회수합니다.

완료 기준:
- 각 티어는 개념 설명보다 상황 진입이 먼저입니다.
- 정답 피드백은 짧고 다음 해금만 암시합니다.
- 철 티어 이전에는 `Good Afternoon.` 네이밍 의도를 직접 설명하지 않습니다.

### 3. Hub: 메인 지도와 진행 상태

역할: 사용자가 지금 어디에 있고, 무엇이 열렸고, 무엇은 아직 잠겨 있는지 보여줍니다.

개발 순서:
1. Pre-Cyan 이후 Cyan으로 이어지는 임시 허브를 만듭니다.
2. 트렁크와 브랜치의 잠금 상태를 한 화면에서 확인하게 합니다.
3. 로컬 저장 포맷을 Pre-Cyan 전용에서 전체 진행 상태로 확장합니다.

완료 기준:
- 진행 상태는 브라우저 새로고침 후 유지됩니다.
- 저장 데이터가 깨져도 초기 상태로 안전하게 복구됩니다.
- 메인 허브는 카드 체크리스트가 아니라 지도와 노드 구조를 기본형으로 씁니다.

### 4. Branch: 투자, 자산, 절세, 습관, 경제현상, 역사

역할: 철 티어 이후 사용자가 관심사별로 깊이를 확장합니다.

개발 순서:
1. 투자 브랜치를 먼저 만듭니다. Pre-Cyan의 복권방/어두운 골목 씨앗과 연결됩니다.
2. 자산 브랜치를 붙여 주거, 보증금, 대출의 생활 맥락을 다룹니다.
3. 나머지 브랜치는 동일한 구조로 확장하되, 각 브랜치에 고유한 체감 상황을 둡니다.

완료 기준:
- 브랜치는 트렁크를 대체하지 않습니다.
- 각 브랜치는 최소 1개의 실생활 판단 상황을 포함합니다.
- 브랜치 해금은 사용자가 “공부할 목록이 늘었다”가 아니라 “갈 수 있는 길이 열렸다”고 느껴야 합니다.

### 5. Mastery: Hard to Master 레이어

역할: 모든 티어를 완료한 뒤에도 끝나지 않는 깊이를 제공합니다. 단순 퀴즈가 아니라 실제 경제 현상과 복합 판단을 다룹니다.

개발 순서:
1. L1: 개념 간 연결 판단
2. L2: 지연 피드백이 있는 복합 문항
3. L3: 뉴스나 금리 시나리오를 읽고 결과를 해석하는 시뮬레이션

완료 기준:
- 점수만 보여주지 않고 사용자의 판단 패턴을 분석합니다.
- 쉬운 진입 경험과 섞지 않습니다. Pre-Cyan에 마스터리 복잡도를 넣지 않습니다.
- 최종 축사와 리포트는 `harness/30_mastery_spec.md`의 철학을 따릅니다.

## 다음 구현 권장 순서

1. Pre-Cyan에서 Cyan 입구 이후의 임시 전환 화면을 추가합니다.
2. Cyan 티어 최소 루프를 정적 앱 하위 모듈로 만듭니다.
3. 전체 진행 상태 저장 스키마를 설계하고 Pre-Cyan 상태와 연결합니다.
4. 메인 허브 지도 초안을 만듭니다.
5. 투자 브랜치의 첫 씨앗을 어두운 골목 발견 상태와 연결합니다.

## 개발 사전 작업

- `AGENTS.md`를 기준 지침으로 사용합니다.
- `GEMINI.md`는 Gemini용 인덱스로 유지합니다.
- 디자인 판단은 `DESIGN.md`의 Good Afternoon 우선 섹션을 먼저 보고, Anthropic 분석은 참고 자료로만 봅니다.
- 구현은 `src/` 아래에서 작은 TypeScript 모듈 단위로 나눕니다. 파일을 열었을 때 바로 책임이 보이는 구조를 유지합니다.
- 로컬 실행은 `npm run dev`, 배포 산출물 확인은 `npm run build`와 `npm run preview`를 기준으로 합니다.
- 검증은 문자열 스캔, `npm test`, `npm run build`, 브라우저 스모크 테스트, 모바일 뷰포트 점검 중 변경 범위에 맞게 수행합니다.
