# Pre-Cyan First Village Design

## 목적

Pre-Cyan은 굿애프터눈의 첫 지역이다. 사용자가 경제를 공부하러 왔다고 느끼기 전에, 작은 게임 월드에 들어와 장소를 둘러보고 만져보며 적응하게 만든다.

핵심 목표는 `Easy to learn`이다. 사용자는 첫 세션에서 경제 용어를 외우거나 문제를 맞히는 것이 아니라, 현실적인 동네 요소를 게임 마을처럼 만지면서 소소한 반응을 얻는다.

## 초기 기획 배경

굿애프터눈은 마인크래프트 업적 보드에서 출발한 경제 개념 잠금해제 서비스다. 다만 이 문서의 범위에서는 `Hard to master`보다 `Easy to learn`을 우선한다.

초기 사용자는 경제에 관심이 없거나, 경제라는 단어 자체를 부담스럽게 느낄 수 있다. 그래서 Pre-Cyan은 문제, 카드, 설명을 앞세우지 않는다. 사용자가 처음 모험을 시작할 때 쉽게 들어오고, 몇 개의 장소를 만지며 적응하고, 소소한 재미를 느끼는 것이 목표다.

이번 설계의 컨셉은 `현실 동네를 닮은 첫 모험 마을`이다. 편의점, 버스정류장, 알바처, 은행, 복권방처럼 익숙한 장소를 쓰지만, 사용 경험은 현실 지도 앱이 아니라 게임 월드와 업적 보드에 가깝다.

## 설계 원칙

- 장소 먼저, 개념 나중.
- 대사보다 짧은 로그.
- 발견 먼저, 보상은 나중.
- 업적은 목표가 아니라 클리어 후 남는 기록.
- 경제 용어, 점수, 진행률, 학습 완료 문구는 첫 경험 전면에 두지 않는다.
- 콘텐츠 문장과 장소별 장면은 베타 유저 피드백을 보고 확장할 수 있게 얇게 시작한다.

## 첫 세션 흐름

1. 둘러보기
   - 사용자는 마을형 지도 화면을 본다.
   - `내 방` 노드만 살아 있고 주변 장소는 흐리게 보인다.
   - 긴 튜토리얼이나 목표 문구는 없다.

2. 만져보기
   - 사용자가 `내 방`을 누른다.
   - 짧은 로그가 나온다.
   - `편의점`, `버스정류장`이 활성화된다.

3. 적응
   - 사용자가 2~3개 장소를 자기 순서대로 누른다.
   - 방문 표시는 남지만 점수, 업적, 진행률은 보이지 않는다.
   - 장소를 누르면 연결된 주변 장소가 하나씩 열린다.

4. 클리어
   - `내 방`을 포함해 공개 장소 4개를 방문하면 `Cyan 입구`가 열린다.
   - 로그는 짧게 유지한다. 예: `길이 열렸다.`
   - 사용자가 Cyan 입구를 누르면 첫 업적을 처음 보여준다.

## 화면 구조

화면은 현실 동네처럼 보이는 요소를 쓰되, 구조는 게임의 업적 보드에 가깝다. 장소 노드는 길처럼 연결되고, 처음에는 일부만 활성화된다.

초기 노드 흐름:

```text
내 방
  -> 편의점
  -> 버스정류장
편의점
  -> 알바처
  -> 구독함
  -> 복권방
버스정류장
  -> 은행
알바처
  -> 은행
은행
  -> Cyan 입구
복권방
  -> 숨김: 복권방 뒤 어두운 골목
```

MVP는 모바일 세로 화면에서 한 화면으로 보는 경험을 1차 기준으로 한다. 모바일 가로 화면은 추후 확장 고려사항으로만 둔다.

## 장소와 로그

장소별 콘텐츠는 개념 설명이 아니라 짧은 로그로 시작한다. 아래 문구는 초기 톤 기준이며, 실제 문장은 베타 피드백에 따라 늘리거나 줄일 수 있다.

| 장소 | 초기 로그 예시 | 숨은 개념 |
| --- | --- | --- |
| 내 방 | `카드를 챙겼다.` | 지출, 결제수단, 기록 |
| 편의점 | `봉투값이 붙었다.` | 가격, 수수료, 세금 |
| 버스정류장 | `잔액이 부족하다.` | 잔액, 선불, 이동비 |
| 알바처 | `입금액이 예상보다 작다.` | 소득, 시급, 공제 |
| 은행 | `숫자가 조금 늘었다.` | 예금, 이자, 금리 |
| 구독함 | `이번 달에도 빠졌다.` | 고정비, 자동결제 |
| 복권방 | `같은 번호가 또 걸려 있다.` | 확률, 기대값, 위험 |
| 복권방 뒤 어두운 골목 | `불이 잠깐 켜졌다.` | 위험자산, 손실, 변동성 |
| Cyan 입구 | `길이 열렸다.` | 본 티어 진입 |

경제 용어는 사용자에게 처음부터 제목처럼 보이지 않는다. 내부 매핑과 추후 콘텐츠 연결을 위한 설계 정보로 관리한다.

## 복권방과 숨은 골목

`복권방`은 공개 노드다. 청소년도 길에서 자주 보는 장소이므로 낯설지 않다. 다만 도박장처럼 직접 표현하지 않고, 운과 확률의 씨앗으로 둔다.

`복권방 뒤 어두운 골목`은 숨은 노드다. 본 진행에는 필요 없고, 발견하지 못해도 끝까지 모를 수 있다.

MVP 발견 조건:

```text
lotterySeen = true
그리고
은행 방문 또는 편의점 재방문
```

발견한 사용자는 나중에 투자 브랜치에서 일부 숨은 노드, 업적, 문항이 먼저 반응할 수 있다. 발견하지 못한 사용자는 일반 투자 브랜치만 본다.

## 상태와 해금

정적 웹 MVP에서는 `localStorage`로 충분하다.

필요 상태:

```text
visitedPlaces
firstHintSeen
cyanGateUnlocked
lotterySeen
backAlleyDiscovered
backAlleyEntered
firstAchievementShown
```

기본 규칙:

- 첫 진입 시 `내 방`만 활성화한다.
- `내 방` 방문 후 `편의점`, `버스정류장`을 활성화한다.
- 장소 방문 시 `visitedPlaces`에 기록한다.
- 연결된 장소를 하나씩 활성화한다.
- `내 방`을 포함해 공개 장소 4개 방문 후 `Cyan 입구`를 활성화한다.
- `Cyan 입구` 진입 시 첫 업적을 표시한다.
- 복권방 뒤 어두운 골목은 별도 발견 조건을 만족할 때만 표시한다.

## 업적 노출 시점

업적은 Pre-Cyan 클리어 직후 처음 보여준다.

첫 진입, 첫 클릭, 적응 단계에서는 업적 보드가 보이지 않는다. 사용자가 업적을 얻기 위해 움직이게 만들지 않는다. 먼저 장소를 만지고, 적응하고, 첫 지역을 통과한 뒤에 기록이 남는 구조로 둔다.

첫 업적 예시:

```text
첫 길
마을 밖으로 나갈 준비가 됐다.
```

업적 문구 역시 짧게 유지한다.

## MVP 포함 범위

- 마을형 업적 보드 화면.
- 시작 노드 `내 방`.
- 공개 장소: 편의점, 버스정류장, 알바처, 은행, 구독함, 복권방.
- 숨김 장소: 복권방 뒤 어두운 골목.
- 장소 클릭 시 짧은 로그 표시.
- 방문 장소 표시.
- 연결 장소 활성화.
- `내 방`을 포함해 공개 장소 4개 방문 후 Cyan 입구 활성화.
- Cyan 입구 진입 시 첫 업적 표시.
- `localStorage` 진행 저장.
- 초기화용 개발자 버튼.

## MVP 제외 범위

- 계정, 서버 저장.
- 전체 107문항 구현.
- 투자 브랜치 본편.
- 복잡한 캐릭터, 인벤토리, 전투 시스템.
- 점수, 랭킹, 결제, 커뮤니티.
- 모바일 가로 화면 상세 설계.

## 기존 문서 정리 방침

이 설계가 승인되면 루트 디렉터리의 기존 기획 문서를 정리할 수 있다. 정리 목적은 문서 수를 줄이는 것이 아니라, Pre-Cyan의 첫 경험 방향과 충돌하는 오래된 표현을 제거하는 것이다.

정리 대상:

- `PROJECT_CONTEXT.md`: Pre-Cyan을 단어 카드 대기실이 아니라 첫 모험 마을로 갱신한다.
- `README.md`: 현재 데모 설명에 Pre-Cyan 첫 지역의 목표와 모바일 세로 기준을 간단히 반영한다.
- `DESIGN.md`: 학습 앱, 카드 중심, 랜딩 페이지 중심으로 읽힐 수 있는 부분이 있으면 게임 월드/업적 보드/짧은 로그 중심으로 조정한다.
- `harness/30_mastery_spec.md`: Pre-Cyan과 K 티어 마스터리의 역할이 섞이지 않도록, Pre-Cyan은 진입 경험, K 티어는 깊이 경험으로 분리한다.

기존 HTML 데모 파일 정리는 별도 구현 단계에서 판단한다. 문서 정리만으로 데모 파일을 삭제하거나 대체하지 않는다.

## 검증 기준

1. 첫 진입
   - 사용자가 설명을 읽지 않아도 첫 노드를 누를 수 있어야 한다.

2. 첫 클릭 후 이해
   - 첫 클릭 뒤 주변 장소가 켜져서 조작 방식을 이해할 수 있어야 한다.

3. 적응
   - 사용자가 2~3개 장소를 자유롭게 눌러도 막히지 않아야 한다.

4. 클리어
   - Cyan 입구가 열리고, 입구를 누른 뒤에야 첫 업적이 표시되어야 한다.

5. 부담감 점검
   - 첫 세션 전면에 경제 용어, 점수, 진행률, 학습 완료 문구가 없어야 한다.
