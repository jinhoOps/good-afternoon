# Pre-Cyan to Cyan Token Transition Design

## Summary

Pre-Cyan에서 Cyan으로 넘어가는 첫 정식 개발 단위는 지도 위 플레이어 기물 이동을 중심으로 설계한다. 현재 Pre-Cyan 구현은 노드가 활성화되고 로그가 바뀌는 방식이라 사용자가 실제로 이동했다는 감각이 약하다. 이 spec은 모노폴리, 문명, 체스처럼 사용자의 말이 지도 위에서 목적지까지 움직이고, Cyan 입구에 도착한 뒤 첫 Cyan 루프가 열리는 경험을 정의한다.

## User Experience Goal

사용자는 버튼을 눌러 상태를 켠 것이 아니라, 자신의 기물이 마을을 지나 Cyan 입구까지 이동했다고 느껴야 한다.

성공한 경험은 다음처럼 읽혀야 한다.

1. 사용자는 내 방 위에 있는 작은 말을 본다.
2. 갈 수 있는 장소를 고른다.
3. 말이 현재 위치에서 선택한 장소로 움직인다.
4. 도착 후 짧은 로그가 나온다.
5. 새 길이 열리고 다음 목적지가 보인다.
6. 공개 장소 4개를 지나면 Cyan 입구가 목적지가 된다.
7. 말이 Cyan 입구에 도착한 뒤 첫 업적과 Cyan 최소 루프가 열린다.

## Scope

### In Scope

- Pre-Cyan 지도 위 플레이어 기물 추가
- 노드 클릭 시 기물이 이동한 뒤 상태가 갱신되는 흐름
- 이동 중 입력 잠금
- 직전 이동 경로 강조
- Cyan 입구를 도착 가능한 목적지로 표현
- Cyan 입구 도착 후 첫 업적 표시
- 첫 Cyan 최소 플레이 루프 1개
- 기존 localStorage 데이터 보강 로직
- 모바일 390x844 기준 레이아웃 검증

### Out of Scope

- 전체 허브 라우터
- Cyan 전체 티어 12문항
- Magenta, Yellow 티어 구현
- 브랜치 77문항 구현
- 사운드, 캐릭터 커스터마이징, 주사위 시스템
- 프레임워크 또는 빌드 도구 도입

## Interaction Model

### Pre-Cyan Map

장소 노드는 목적지다. 사용자의 존재감은 노드 스타일이 아니라 별도 플레이어 기물이 담당한다.

- 기물은 초기 상태에서 `room` 위에 있다.
- 잠금 해제된 노드만 목적지로 선택할 수 있다.
- 사용자가 목적지를 선택하면 기물이 현재 노드에서 목표 노드로 이동한다.
- 이동 중에는 모든 노드 입력을 비활성화한다.
- 이동이 끝난 뒤에 방문 처리, 로그 변경, 다음 노드 해금이 일어난다.
- 현재 위치 노드는 시각적으로 더 선명하게 표시한다.
- 방금 지나간 경로는 짧게 강조한다.

### Cyan Gate

Cyan 입구는 조건 충족 후 나타나는 버튼이 아니라, 기물이 도착할 수 있는 목적지다.

- 공개 장소 4개 방문 후 `cyanGate`가 이동 가능한 노드가 된다.
- 사용자가 `cyanGate`를 선택하면 기물이 입구까지 이동한다.
- 도착 후 Pre-Cyan 첫 업적이 표시된다.
- 업적 표시 이후 Cyan 최소 루프 패널이 열린다.

### Cyan Minimal Loop

첫 Cyan 루프는 “화폐와 교환”을 강의하지 않는다. Pre-Cyan에서 이미 본 생활 사건을 이어받아, 사용자가 교환 관계를 한 번 판단하게 한다.

첫 루프 예시:

- 상황: 버스정류장에서 잔액이 부족하다.
- 질문 톤: “지금 오가야 하는 건 뭐지?”
- 선택지:
  - 교통카드 잔액을 내고 이동 시간을 얻는다.
  - 이동 시간을 내고 봉투값을 얻는다.
  - 복권 번호를 내고 알바 시간을 얻는다.
- 성공 로그: “맞아. 뭔가를 내고, 길을 얻었다.”
- 실패 로그: “그건 아직 길이 안 이어진다.”

점수, 학습 완료, 긴 개념 설명은 첫 Cyan 루프에서 쓰지 않는다.

## State Model

기존 Pre-Cyan 상태에 다음 필드를 추가한다.

```javascript
{
  playerNodeId: 'room',
  movingToNodeId: null,
  lastMove: null,
  currentStage: 'preCyan',
  cyanLoopSeen: false,
  cyanLoopCompleted: false
}
```

필드 의미:

- `playerNodeId`: 기물이 현재 서 있는 노드 id
- `movingToNodeId`: 이동 중 목표 노드 id, 이동 중이 아니면 `null`
- `lastMove`: 직전 이동 경로. 예: `{ from: 'room', to: 'store' }`
- `currentStage`: `preCyan`, `cyanIntro`, `cyanLoop` 중 하나
- `cyanLoopSeen`: Cyan 최소 루프를 한 번 열었는지 여부
- `cyanLoopCompleted`: 첫 Cyan 판단을 완료했는지 여부

기존 저장 데이터에는 이 필드가 없을 수 있다. `loadState()`는 누락 필드를 초기값으로 보강해야 한다. 저장 데이터가 깨졌을 때는 기존처럼 초기 상태로 안전하게 복구한다.

## Rendering Model

### Map Layer

지도는 기존 노드와 경로를 유지하되, 플레이어 기물 레이어를 추가한다.

- 경로 SVG
- 노드 버튼 레이어
- 플레이어 기물 레이어
- 로그와 업적/루프 패널

기물은 노드 좌표를 그대로 사용한다. 이동은 CSS transition으로 처리한다. 구현은 단순해야 하며, 애니메이션 엔진을 도입하지 않는다.

### Input Locking

이동 중에는 다음을 보장한다.

- 노드 클릭이 무시된다.
- 중복 방문 처리나 중복 로그 업데이트가 일어나지 않는다.
- 이동 종료 이벤트 이후 한 번만 상태가 갱신된다.

CSS transition 종료 이벤트가 누락될 가능성을 대비해 짧은 timeout fallback을 둔다. fallback은 같은 상태 갱신 함수를 재사용해야 하며, 중복 실행을 막는 guard를 둔다.

## Data Boundaries

기존 파일 책임은 유지한다.

- `village-data`: Pre-Cyan 노드와 경로, Cyan 첫 루프 데이터
- `village-state`: 저장, 이동 시작/완료, 방문/해금 규칙, Cyan 루프 완료 규칙
- `village-render`: DOM 렌더링, 클릭 이벤트, 이동 종료 이벤트 연결
- `styles.css`: 기물, 이동 상태, Cyan 루프 패널 스타일

첫 구현에서는 별도 앱을 만들지 않는다. `app/cyan-tier/` 분리는 전체 허브와 티어 구조가 필요해지는 시점으로 미룬다.

## Error Handling

- 저장 데이터 파싱 실패: 초기 상태로 복구한다.
- 알 수 없는 노드 선택: 상태를 변경하지 않는다.
- 잠긴 노드 선택: 상태를 변경하지 않는다.
- 이동 중 노드 선택: 상태를 변경하지 않는다.
- transition 종료 이벤트 누락: timeout fallback으로 이동 완료 처리한다.
- fallback과 transition 종료가 둘 다 발생: guard로 이동 완료를 한 번만 처리한다.

## Accessibility

- 노드 버튼은 기존 button semantics를 유지한다.
- 플레이어 기물은 시각 장식이므로 `aria-hidden="true"`로 둔다.
- 현재 위치 정보는 로그 또는 상태 텍스트로 전달한다.
- 이동 중에는 board에 `aria-busy="true"`를 적용한다.
- Cyan 루프 선택지는 버튼으로 제공한다.

## Verification Criteria

구현 완료 후 다음을 확인해야 한다.

1. 초기 로드 시 기물이 `내 방` 위치에 있다.
2. `내 방` 클릭 또는 초기 상태에서 갈 수 있는 노드가 명확히 보인다.
3. 잠금 해제된 노드를 클릭하면 기물이 목표 노드로 이동한다.
4. 이동이 끝난 뒤 방문 상태, 로그, 다음 해금이 갱신된다.
5. 이동 중 다른 노드를 눌러도 상태가 중복 갱신되지 않는다.
6. 공개 장소 4개 방문 후 Cyan 입구가 도착 가능한 목적지가 된다.
7. Cyan 입구를 클릭하면 기물이 Cyan 입구로 이동한다.
8. Cyan 입구 도착 후 첫 업적이 표시된다.
9. 업적 이후 Cyan 최소 루프가 표시된다.
10. Cyan 최소 루프의 성공/실패 로그는 점수형 표현을 쓰지 않는다.
11. 새로고침 후 기물 위치, 방문 상태, Cyan 진행 상태가 유지된다.
12. 초기화 버튼으로 기물 위치와 Cyan 상태가 초기화된다.
13. 모바일 390x844에서 기물, 노드, 로그, 선택지가 겹치지 않는다.
14. 금지 문자열 `필수 기초 단어 놀이터`, `이해함`, `점수`, `학습 완료`가 첫 경험 코드에 나타나지 않는다.

## Open Decisions Resolved

- 플레이어 존재감은 노드 스타일이 아니라 별도 기물로 표현한다.
- Cyan 입구는 화면 전환 버튼이 아니라 이동 가능한 목적지로 표현한다.
- 첫 Cyan 구현은 별도 앱이 아니라 현재 Pre-Cyan 앱 안에서 확장한다.
- Cyan 전체 티어는 이 spec 범위에 넣지 않는다.
- 이동감은 CSS transition으로 구현하고, 외부 애니메이션 라이브러리는 쓰지 않는다.
